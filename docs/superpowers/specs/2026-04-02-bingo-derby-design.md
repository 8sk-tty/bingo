# Bingo Derby - 設計仕様書

## Context

イベント・宴会向けのビンゴ抽選Webアプリ。プロジェクターや大画面に映して使う想定。
単なる数字抽選ではなく、競馬・パチンコ・スロットなどの演出をランダムに挟むことで場を盛り上げる。
Vercelにデプロイする。

## 要件サマリー

- 1〜75の標準ビンゴ（B/I/N/G/O列）
- 大画面投影用の1ページ構成
- 抽選トリガー: ボタンクリック + スペースキー
- 約30%の確率でランダムに特別演出が発動（それ以外はシンプルな数字表示）
- B-I-N-G-Oボードで抽選済み番号を常時表示
- デザイン: 深紺背景 × ゴールドアクセント（カジノ風ラグジュアリー）
- やり直し/リセット機能は不要

## 技術スタック

- **Next.js 16** (App Router)
- **TypeScript**
- **Framer Motion** — 全アニメーション
- **Tailwind CSS** — スタイリング
- **Vercel** — デプロイ
- **状態管理**: React useState（サーバー通信なし、1ページ完結）

## 抽選フロー

1. DRAWボタン押下（クリック or スペースキー）
2. 未抽選の番号からランダムに1つ選択
3. 演出判定: 約30%の確率で特別演出を発動
   - **演出なし（デフォルト）**: 数字がぼやけた状態から徐々にフォーカス → スケール小→大にアニメーション → ゴールドの光パーティクルが散る（約1.5秒）
   - **演出あり**: 6種類からランダムに1つ選出 → 演出再生 → 数字確定表示
4. ボードの該当番号をゴールドにハイライト
5. DRAWボタンが再び有効に

## 演出一覧

| # | 名前 | 定数キー | 概要 |
|---|------|----------|------|
| 1 | 競馬 | HORSE_RACE | 数字をつけた5頭の馬が左→右に走り、1位ゴールの馬の数字が当選 |
| 2 | パチンコ | PACHINKO | 玉が釘に当たりながら落下 → V字ゲートに入って大当たり → 数字確定 |
| 3 | スロット | SLOT_MACHINE | 3つのリールが回転 → 順番に停止 → 3つ揃って数字確定 |
| 4 | ルーレット | ROULETTE | ルーレット盤回転 → ボールが減速 → ポケットに落ちて数字確定 |
| 5 | ガラポン | GARAPON | ガラポン機回転 → 玉がコロコロ出る → 番号確定 |
| 6 | マリオブロック | MARIO_BLOCK | ブロック表示 → 叩くアニメ → コインが飛び出して数字が現れる |

**共通仕様:**
- 演出中はDRAWボタン無効化
- 演出完了後、確定した数字が大きく表示される（共通NumberRevealアニメーション）
- 効果音はなし（将来追加可能な構造にする）

## 定数設計 (`constants/config.ts`)

全てのマジックナンバーを定数として一元管理する。

```ts
// 演出の持続時間（ミリ秒）
export const EFFECT_DURATION = {
  HORSE_RACE: 5000,
  PACHINKO: 5000,
  SLOT_MACHINE: 4000,
  ROULETTE: 5000,
  GARAPON: 4000,
  MARIO_BLOCK: 3000,
} as const;

// 演出なし時のシンプル表示時間（ミリ秒）
export const SIMPLE_REVEAL_DURATION = 1500;

// 演出が発動する確率（0〜1）
export const EFFECT_TRIGGER_PROBABILITY = 0.3;

// ビンゴ数字範囲
export const BINGO_RANGE = { MIN: 1, MAX: 75 } as const;

// B-I-N-G-O列の範囲定義
export const BINGO_COLUMNS = {
  B: { min: 1, max: 15 },
  I: { min: 16, max: 30 },
  N: { min: 31, max: 45 },
  G: { min: 46, max: 60 },
  O: { min: 61, max: 75 },
} as const;
```

## デザイン仕様

- **背景**: 深紺 (`#0a0e27`)
- **メインアクセント**: ゴールド (`#d4af37`)
- **セカンダリ**: 明るいゴールド (`#f5d77a`)
- **テキスト**: ホワイト (`#ffffff`)
- **ボードのハイライト**: ゴールドグロウ（抽選済み番号）
- **フォント**: 大画面向けの大きなサイズ、数字は等幅フォント推奨
- **レスポンシブ**: 大画面優先だがモバイルでも崩れない程度に対応

## 演出コンポーネント共通インターフェース

```ts
interface EffectProps {
  targetNumber: number;  // 当選番号
  onComplete: () => void; // 演出完了時のコールバック
}
```

全演出コンポーネントはこのインターフェースを実装し、演出終了時に`onComplete`を呼ぶ。

## 検証方法

1. `npm run dev`でローカル起動
2. DRAWボタンのクリック/スペースキーで抽選が動作すること
3. 複数回抽選して演出がランダムに発動すること
4. 全6種類の演出が正しく表示され、数字が確定すること
5. ボードに抽選済み番号が反映されること
6. 75回抽選したら全番号が出尽くすこと
7. `vercel deploy`でデプロイ確認
