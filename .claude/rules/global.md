# グローバルルール

## 言語・コミュニケーション
- 必ず日本語で会話すること
- 会話の中で「完璧」や「絶対」という言葉を使わない
  - 事実ベースで会話し、情報の信頼度を数値で明記すること
- 過度にへりくだらない。ユーザも間違えることがあるため、自信を持って発言する
- イレギュラー（想定外の問題）が発生した場合、必ず作業を中断して報告すること

## パッケージマネージャー
- **pnpm のみ使用、npm は禁止**
- `npm install`, `npm run` 等は一切使用しない

## 開発原則
- Next.js 16 のベストプラクティスに準拠すること
- ライブラリやフレームワークは最新の公式ドキュメントを参照して実装
- 実装完了後は動作確認を行い、正しく実装されたことを確認
- ボーイスカウトの精神: 実装中に発見した小さなバグや間違いは報告しつつ修正する
  - commit は必ず分離する
  - タスクの主目的から逸脱する場合は TODO コメントだけ記載する

## 保守性
- 機能追加と同時に既存コードの改善を検討
- 大規模な変更は小さなステップに分割
- 使用されていないコードは積極的に削除
- 依存関係は定期的に更新（セキュリティと互換性のため）
- 技術的負債は明示的にコメントやドキュメントに記録
- 本当に必要な依存関係のみを追加
- 新しい依存関係追加前にライセンス、サイズ、メンテナンス状況を確認

## トレードオフの認識
- 制約の中で最適なバランスを見つける
- プロトタイプなら簡潔さを、本番なら堅牢性を優先
- 妥協点とその理由を明確にドキュメント化
- ビジネス要件と技術要件のバランスを取る
- 現在のフェーズで本当に必要な品質レベルを判断
