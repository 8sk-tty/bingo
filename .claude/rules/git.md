# Git運用ルール

## コンベンショナルコミット
コミットメッセージは以下の形式を使用:
- `feat:` - 新機能
- `fix:` - バグ修正
- `docs:` - ドキュメント
- `test:` - テスト
- `refactor:` - リファクタリング
- `chore:` - 雑務

## コミットの原則
- コミットは原子的で、単一の変更に焦点を当てる
- 明確で説明的なコミットメッセージを英語で記述
- main/master ブランチへの直接コミットは避ける
- ボーイスカウト修正は必ず別コミットに分離する

## ブランチ運用
- メインブランチ: `main`
- ステージングブランチ: `stg`（統合テスト環境用）
- 機能ブランチ: `feature/xxx`
- 修正ブランチ: `fix/xxx`
- pnpm-lock.yaml 等のロックファイルを必ずコミット

### ブランチの起点とマージ先
- 作業ブランチは必ず `main` から切る（`stg` から切らない）
- PRは `main` に向けて出す
- `stg` には作業ブランチや `main` をマージして統合テストを行う
- `stg` が汚れた場合は `main` から再作成する
