# Git運用ルール

## コンベンショナルコミット
コミットメッセージは以下の形式を使用:
- `feat:` - 新機能
- `fix:` - バグ修正
- `docs:` - ドキュメント
- `test:` - テスト
- `refactor:` - リファクタリング
- `chore:` - 雑務

## コミットの原則
- コミットは原子的で、単一の変更に焦点を当てる
- 明確で説明的なコミットメッセージを英語で記述
- main/master ブランチへの直接コミットは避ける
- ボーイスカウト修正は必ず別コミットに分離する

## ブランチ運用（GitHub Flow）
- メインブランチ: `main`
- 機能ブランチ: `feature/xxx`
- 修正ブランチ: `fix/xxx`
- pnpm-lock.yaml 等のロックファイルを必ずコミット
- マージ後のブランチは GitHub が自動削除（delete-branch-on-merge 有効）

### ブランチの起点とマージ先
- 作業ブランチは必ず `main` から切る
- PR は `main` に向けて出す
- 検証は Vercel の PR プレビューデプロイを利用する
