# /review - コードレビュー

現在の変更に対して `senior-code-reviewer` エージェントを使ったコードレビューを実施します。

## 手順

1. `git diff` で変更内容を取得（引数があればそのファイル/ブランチを対象）
2. `senior-code-reviewer` サブエージェントに変更内容を渡してレビューを依頼
3. レビュー結果を以下の形式で報告:
   - **重要度: 高** - 必ず修正が必要な問題（セキュリティ、バグ、型安全性）
   - **重要度: 中** - 修正推奨（パフォーマンス、保守性）
   - **重要度: 低** - 改善提案（命名、コードスタイル）
4. 修正が必要な項目について、ユーザーに修正するか確認

## レビュー観点

- TypeScript型安全性
- エラーハンドリング
- パフォーマンス
- プロジェクトのコーディング規約準拠
- コンポーネント設計の妥当性
