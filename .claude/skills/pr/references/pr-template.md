# PR 本文テンプレート

PR 作成時に使用する本文のテンプレート。

## テンプレート

```markdown
## Summary

- <変更の概要を 1-3 行で記述>

## Changes

- <変更点 1>
- <変更点 2>
- <変更点 3>

## Test Plan

- [ ] <テスト項目 1>
- [ ] <テスト項目 2>
- [ ] <テスト項目 3>
```

## 各セクションの記述ガイド

### Summary

- 変更の目的と背景を簡潔に記述する
- 関連 Issue がある場合は `Closes #<issue-number>` を含める
- 技術的な詳細は Changes セクションに記述する

### Changes

- 変更内容を箇条書きで具体的に記述する
- ファイル単位ではなく、機能単位でグルーピングする
- 破壊的変更がある場合は明示する

### Test Plan

- テスト方法をチェックリスト形式で記述する
- 手動テストの手順がある場合は含める
- 自動テストの実行コマンドがある場合は記載する
