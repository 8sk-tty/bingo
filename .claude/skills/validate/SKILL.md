# /validate - 全品質チェック統合

TypeScript型チェック、リント、テストを順次実行し、結果を分析・報告します。

## 手順

1. **TypeScript型チェック**: `pnpm build` を実行（型エラーがあればビルド失敗で検出）
   - エラーがあれば一覧を表示し、修正が必要な箇所を特定
2. **ESLint**: `pnpm lint` を実行（ESLint 導入済みの場合）
   - エラーと警告を分類して報告
   - 未導入の場合はスキップ
3. **ユニットテスト**: `pnpm test` を実行（テストフレームワーク導入済みの場合）
   - 失敗したテストがあれば詳細を表示
   - 未導入の場合はスキップ
4. 全結果のサマリーを報告:
   - パス / 失敗 の一覧
   - 失敗した項目の修正提案

## 結果のフォーマット

```
## バリデーション結果

| チェック | 結果 | 詳細 |
|----------|------|------|
| TypeScript型チェック（build） | パス / 失敗 | エラー数 |
| ESLint | パス / 失敗 / 未導入 | エラー数 / 警告数 |
| ユニットテスト | パス / 失敗 / 未導入 | パス数 / 失敗数 |
```

## 注意

- 各ステップは前のステップが失敗しても続行する
- 修正の自動適用は行わず、報告のみ
